# /nabu-metrics

Работа с метриками жизни (фича 13). Разбери `$ARGUMENTS`:

- **`log <метрика> <значение> [ед.] [домен]`** — записать значение: `nabu-domain.log_metric`.
- **`<метрика>`** (без log) — показать тренд/агрегаты/аномалии: `nabu-analytics.aggregate_metric` +
  `forecast_metric` + `detect_anomalies`.
- **`correlate <домен>`** или без аргументов — запусти субагент `outcome-analyst`: связать советы
  Nabu по сферам с динамикой метрик (`nabu-analytics.correlate_metrics`), честно (ассоциация, не
  причинность; p-value, n, конфаундеры). Записать `record_effectiveness` (source=outcome), инсайты — в `60-metrics/`.

Приватные метрики (здоровье/финансы) — локально, не в облако/логи. docs/28: с метриками
настроения/здоровья бережно, без тревожных выводов из шума. По расписанию — /nabu-cron (ежедневно).
