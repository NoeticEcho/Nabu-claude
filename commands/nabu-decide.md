# /nabu-decide

Запусти decision-maker по `$ARGUMENTS`: формулировка → критерии и веса → альтернативы →
оценка (MCDA/expected utility) → анализ рисков → проверка когнитивных искажений →
рекомендация с прозрачной логикой. Используй память (прошлые решения, ценности, исходы).
Финальное решение — за пользователем. При многодоменности — сначала собрать позиции Совета.
