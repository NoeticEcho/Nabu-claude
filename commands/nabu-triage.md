# /nabu-triage

Разбери «Входящие» пользователя: свежие заметки (`fleeting`) и файлы из `$NABU_HOME/00-inbox`.
Каждый элемент — классифицировать (сфера/тип/визибилити), извлечь сущности и устойчивые факты
в память/граф, задать статус и разложить по разделам workspace. Тяжёлое (эмбеддинги/индексация)
выполняется ЛОКАЛЬНО серверами `nabu-pipeline`/`nabu-memory` — приватный контент не проходит через Claude.

## Шаги

1. **Собрать входящие.** `nabu-pipeline.list_notes({ status: "fleeting", limit: 20 })` +
   `nabu-pipeline.scan_folder({ path: "$NABU_HOME/00-inbox" })` (верхний уровень). **Бюджет:**
   ≤20 элементов суммарно за один прогон — остальное честно отложи и сообщи, сколько осталось.

2. **Для КАЖДОЙ заметки:**
   - Классифицируй: **сфера** (health, mind, finance, work, learning, relationships, growth,
     lifestyle, admin), **тип** (`fleeting` | `literature` | `evergreen` | `task` | `idea`),
     **visibility** — медицина/финансы/отношения → всегда `private` (нет данных → считать `private`).
   - Субагентом `entity-extractor` извлеки сущности/связи и устойчивые факты → пиши через
     `nabu-memory.graph_upsert_concept` / `nabu-memory.graph_relate` (граф) и `nabu-memory.add_fact`
     (semantic_facts). Не выдумывать — только из текста.
   - Если это задача/намерение → `nabu-memory.add_prospective` (или доменная запись через
     `nabu-domain`). Это **write-класс** — сообщи пользователю, не молча.
   - Затем `nabu-pipeline.update_note({ id, status: <новый>, type, domain })`.

3. **Для файлов 00-inbox:** прочитать (`Read`), тот же разбор, что и для заметок. Проиндексировать
   `nabu-pipeline.index_document`. Предложить/выполнить перенос файла в подходящий раздел workspace
   (перемещение файлов делает адъютант; `PostToolUse`-хук закоммитит изменение).

4. **Итог.** Краткая сводка: сколько разобрано и куда, сколько сущностей/фактов добавлено, что
   отложено (и сколько осталось). Записать эпизод `nabu-memory.remember_episode` (visibility `private`).

## Vault-заметки (особый путь)
Содержимое vault-заметок в `list_notes` приходит как `[vault: скрыто]` — НЕ пытайся его читать.
Экстракция — только локальной моделью, минуя контекст Claude:
`nabu-pipeline.extract_entities_local({ noteId })` → сервер сам расшифрует, извлечёт сущности/факты
локальной LLM и вернёт ТОЛЬКО их. Классифицируй такие заметки по title + извлечённым сущностям;
факты в память пиши с visibility `vault`.

## Инварианты
- Не выдумывать факты о пользователе — только из текста заметок/файлов.
- Приватность: по умолчанию `private`; private/vault не уходят в облако/Claude/логи.
- Бюджет ≤20 элементов за прогон — остаток отложить и сообщить.
- При сомнительном/чувствительном — привлечь `critic` перед записью.
