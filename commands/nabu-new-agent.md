# /nabu-new-agent

Создай агента для `$ARGUMENTS`. Проектирование делает субагент `agent-creator`, а файлы пишет
адъютант (у субагентов нет Write/Edit — least-privilege).

1. **Проверка дубля** — адъютант/agent-creator сверяются с `agents/registry.json`: нет ли уже
   подходящего министра/специалиста. Дубли не создаём.
2. **Проектирование (`agent-creator`, субагент)** — определяет роль, место (под министерством),
   класс риска; подбирает профиль личности (`PERSONALITY_RENDERING.md`) и guardrails; прописывает
   границы компетенции (`SAFETY.md` для high-risk); готовит **полный текст** дефиниции
   `agents/<slug>.md` (frontmatter + тело + «Режим команды»), JSON профиля, запись реестра и eval-набор.
   Возвращает всё адъютанту (сам файлы не пишет).
3. **Ревью (`critic`, субагент)** — адъютант отдаёт артефакты критику: границы, guardrails,
   отсутствие broad-инструментов, SAFETY/privacy. Без прохождения — не активировать.
4. **Запись (адъютант)** — после ревью адъютант пишет `agents/<slug>.md` + `agents/<slug>.json`
   и добавляет запись в `agents/registry.json` (`spawned`/специалист министерства, `impl: "agent:<slug>"`).

Не создавать агентов, обходящих safety, и не плодить впрок.
