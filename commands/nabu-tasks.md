# /nabu-tasks

Ведение дел пользователя (nabu-domain). По `$ARGUMENTS`:

1. **Пусто / «обзор»** — картина дня: `list_tasks({open: true, dueWithinDays: 1})` (просроченные
   и сегодняшние —первыми, пометь ⚠ просроченные), затем остальные открытые (`open: true`,
   ≤10). Сгруппируй по проектам, покажи id-префиксы для закрытия.
2. **«неделя»** — `list_tasks({open: true, dueWithinDays: 7})` + открытые без срока (кратко).
3. **«добавь …»** — распарси текст: заголовок, срок (сегодня/завтра/дата → YYYY-MM-DD),
   приоритет (срочно/важно → high) → `add_task`. Несколько строк = несколько задач.
   Подтверди списком созданного.
4. **«закрой …» / «сделал …»** — найди задачу по подстроке в открытых → `update_task_status(id,
   "done")`. Неоднозначно — покажи кандидатов, спроси. Закрытие важной задачи → предложи
   `award_xp` (discipline) с причиной.

Быстрый ввод без Claude: в Telegram любая строка `!текст [@завтра|@YYYY-MM-DD]` создаёт задачу
мгновенно. Задачи со сроком попадают в утренний брифинг автоматически.
Не выдумывать задачи; не закрывать без явного указания пользователя.
