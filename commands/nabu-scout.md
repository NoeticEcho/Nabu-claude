# /nabu-scout

Запусти субагент `capability-scout` (фича 11) для `$ARGUMENTS`. Он:
1. Ищет в вебе/GitHub (`WebSearch`/`WebFetch`/`gh`) под потребность: скиллы, MCP-серверы, скрипты,
   плагины, приёмы; проверяет источник/лицензию/безопасность/свежесть.
2. Оценивает пригодность к архитектуре Nabu (субагенты/skills/MCP, приватность, узкие tools, harness).
3. Оформляет находки `nabu-improve.add_proposal` (category skill|mcp|tool) со ссылками и рисками.
4. Если просят «напиши скилл для X» — черновик SKILL.md по канону → в `90-system/proposals/`;
   активация через `agent-creator` + `critic`.

**Ничего не устанавливает и не исполняет без approval** (`request_approval`). Только реально
найденное (со ссылками). Приватные данные в поиск не отправляются. По расписанию — см. `/nabu-cron`.
