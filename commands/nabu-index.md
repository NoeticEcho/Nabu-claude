# /nabu-index

Построй базу знаний из папки `$ARGUMENTS`, используя узкие типизированные tools серверов
`nabu-pipeline` и `nabu-memory`. Тяжёлое (чтение/чанкинг/эмбеддинги) выполняется ЛОКАЛЬНО
на сервере — приватный контент не проходит через Claude.

## Шаги

1. **Скан** — `nabu-pipeline.scan_folder({ path })`. Показать, сколько файлов найдено и что
   пропущено. При большом объёме предупредить о бюджете и подтвердить.

2. **Визибилити** — по умолчанию `private` (локально). Медицина/финансы/отношения → всегда `private`.

3. **Индексация (локально)** — `nabu-pipeline.index_folder({ path, visibility })`. Сервер читает
   файлы, чанкует, считает эмбеддинги (Ollama, 768) и пишет в базу знаний. Прогресс в stderr
   (`[nabu-index] <файл>: N чанков`). Идемпотентно по файлу.

4. **Анализ в сущности/факты/граф (по умолчанию, не опционально)** — чтобы заметки были
   ПРОАНАЛИЗИРОВАНЫ, а не просто разбиты на чанки. Для проиндексированных документов подними
   фрагменты (`search_knowledge`) и субагентом `entity-extractor` (при необходимости + `linker`,
   `domain-classifier`) извлеки: сущности/связи → `nabu-memory.graph_upsert_concept` / `graph_relate`
   (граф TypeDB), устойчивые факты → `nabu-memory.add_fact` (semantic_facts). Не выдумывать — только
   из текста. **Бюджет:** при большом объёме обрабатывай батчами/приоритетно ключевые документы и
   сообщи, сколько осталось (не тихо усекать). Веди счётчик добавленных концептов/связей/фактов.
   > Примечание: эпизодическая память — про взаимодействия с пользователем, не про индексируемые
   > документы; при индексации она НЕ пишется (эпизоды появляются в диалоге через `/nabu-ask`).

5. **Проверка приватности** — private/vault не уходят в облако/Claude/логи. Эмбеддинги — локальные.
   Путь вне разрешённых корней (`NABU_INDEX_ROOTS`, по умолч. `$NABU_HOME`+home) сервер отклонит.

6. **Итог** — раздельно: `nabu-pipeline.knowledge_stats` даёт **{документы, чанки}** (база знаний);
   концепты/связи/факты — из ТВОЕГО счётчика на шаге 4 (в графе/семантике; отдельного tool-счётчика
   по ним нет). База готова для `/nabu-ask`, `/nabu-council`, `/nabu-recall`.

## Бюджет и границы
- Лимит сканирования — 500 файлов, ≤2МБ на файл (сервер усечёт и предупредит).
- Ошибки чтения не прерывают процесс — возвращаются в поле `failed`.
