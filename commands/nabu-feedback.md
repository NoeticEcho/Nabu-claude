# /nabu-feedback

Запусти субагент `feedback-collector` (фича 13) для `$ARGUMENTS` (или по всем ожидающим):

1. `nabu-improve.list_recommendations_followup` — советы, по которым время узнать исход (взять 1-3).
2. Бережно спроси: применил (полностью/частично/нет), что вышло, оценка результата. Без осуждения.
3. `nabu-improve.record_outcome({ id, status, outcome, outcomeRating, sourceAgent, domain })` —
   обновит рекомендацию и запишет эффективность (source=feedback), замыкая контур само-улучшения.
4. Если пользователь делится измеримым — `nabu-domain.log_metric({ name, value, unit?, domain? })`.

Бережно и не назойливо (docs/28): при дистрессе — ресурсы поддержки, не сбор данных. Приватное —
локально. Только со слов пользователя — не выдумывать исходы. По расписанию — /nabu-cron (2-3×/нед).
