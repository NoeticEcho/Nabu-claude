# /nabu-ask

Обработай вопрос `$ARGUMENTS` через адъютанта (skill `nabu-orchestrator`):
1. Триаж (простой / единодоменный / многодоменный).
2. Поднять релевантную память (retriever).
3. Ответить сам, привлечь министра или созвать Совет (`/nabu-council` логика).
4. Для значимого/высокорискового — critic перед выдачей.
5. Записать эпизод/факты/намерения в память. Если дал значимый совет/рекомендацию (особенно из
   Совета) — зафиксировать `nabu-improve.record_recommendation({ sourceAgent, domain, question,
   recommendation, followUpAt })` для последующего трекинга исхода (`/nabu-feedback`).
Соблюдать privacy (private/vault локально) и границы компетенции (SAFETY.md).
