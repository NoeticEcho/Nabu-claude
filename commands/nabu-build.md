# /nabu-build

Запусти созидание через субагент-оркестратор `builder` для `$ARGUMENTS`:

1. Поднять контекст (`nabu-memory.recall` / субагент `retriever`): прошлые проекты, цели, стек, бренд.
2. Стратегия/валидация — субагент `entrepreneurship` (бизнес-модель, гипотезы, GTM; без гарантий).
3. Продукт/технологии — субагент `software-dev` (архитектура, стек, реализация; деплой — через approval).
4. Сайт — субагент `web-dev` (преимущественно Hugo: структура, тема, контент, локальная сборка).
5. Тексты — субагент `copywriting` (value prop, заголовки, CTA, tone of voice; честно).
6. При компромиссах сфер — Совет (`council`); при выборе — `decision-maker`.
7. `critic` перед выдачей значимого; запись в память (`memory-keeper`).

Высокорисковое (деплой, покупка домена/хостинга, отправка писем, финобязательства, регистрация
юрлица) — только через approval (`nabu-memory.request_approval` → согласие пользователя → `log_action`).
Не заменяет юриста/бухгалтера/финсоветника. Решение и запуск — за пользователем. Бюджет: не
разворачивать полную оркестрацию на мелкую задачу — отдай её нужному субагенту напрямую.
