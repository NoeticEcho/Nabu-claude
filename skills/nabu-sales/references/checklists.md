# Чек-листы

## Перед outbound-кампанией
- [ ] Список 50 точных ICP (найдены реально, не «примерно такие»)
- [ ] Гипотеза их боли одним предложением — из памяти/discovery, не выдумана
- [ ] Персонализация в 3 строки: триггер про них / боль / маленький CTA
- [ ] Каденция 3 касаний с выходом; без давления и fake scarcity (Iron Law 1)
- [ ] Никаких выдуманных кейсов/цифр результатов (Iron Law 4)
- [ ] Отправка от лица пользователя → через `request_approval`, не автономно
- [ ] Контакты и переписка пойдут в память как `private`

## Перед discovery-звонком
- [ ] Изучен контекст клиента (чтобы не тратить его время на Situation)
- [ ] Заготовлены вопросы P/I/N (проблема/последствия/ценность), не питч
- [ ] План слушать ~70%; демо только ПОСЛЕ и под выявленную боль
- [ ] Готов честно дисквалифицировать, если не подходит

## После звонка
- [ ] Записан итог и реальные слова клиента о боли (`nabu-domain`, `private`)
- [ ] Квалификация обновлена: боль / бюджет / срок / решатель
- [ ] **Конкретный следующий шаг с датой** зафиксирован (нет шага → сделка стоп)
- [ ] Обещанный follow-up — задача со сроком (CRM-гигиена, Iron Law 5)
- [ ] Стадия пайплайна обновлена честно (в т.ч. lost)

## Еженедельная ревизия пайплайна
- [ ] У каждой активной сделки есть next step с датой; просроченные — разобрать
- [ ] Сделки без движения дольше срока → пометить lost, не раздувать иллюзию
- [ ] Конверсии между стадиями — где течёт? (по реальным данным)
- [ ] Что дисквалифицировать честно и отпустить (освободить фокус)
- [ ] Выводы — эпизодом в память; повторяющиеся возражения — на заметку продукту
