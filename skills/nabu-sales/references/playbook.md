# Плейбук: outbound → discovery → квалификация → возражения → пайплайн

## 1. Outbound для solo (качество, не объём)

1. **Список 50 ICP**: узкий сегмент, которого реально можно найти и до кого дотянуться. Не 500
   размытых — 50 точных. Критерий из памяти (ICP через `retriever`), не «все, у кого есть боль».
2. **Персонализация в 3 строки**: (а) конкретный триггер про НИХ (пост/событие/контекст), не
   «увидел ваш профиль»; (б) гипотеза их боли одним предложением; (в) один маленький вопрос/CTA,
   не «созвон на 30 минут». Без манипуляций и фальшивой лести (Iron Law 1).
3. **Каденция 3 касаний**: касание 1 — короткое ценностное; касание 2 (+3-4 дня) — другой угол/
   польза, не «просто поднимаю вверх»; касание 3 (+5-7 дней) — вежливый выход («закрываю тему, если
   не актуально»). После — стоп. Давить дальше = спам.

## 2. Discovery (SPIN, сжато)

Цель — чтобы клиент сам сформулировал боль и её цену, а не вы её ему продали. Слушать ~70% времени.

| Тип | Смысл | Пример |
|---|---|---|
| **S**ituation | контекст, кратко (не допрос) | «как сейчас устроен процесс X?» |
| **P**roblem | где болит | «что в этом сложнее всего?» |
| **I**mplication | цена проблемы | «во что это выливается по времени/деньгам?» |
| **N**eed-payoff | ценность решения его словами | «если бы это ушло, что изменилось бы?» |

Правило: демо ПОСЛЕ discovery и только под выявленную боль. Демо вслепую = презентация в пустоту.

## 3. Квалификация (бюджет / боль / срок / решатель)

- **Боль**: осознана и приоритетна? «было бы неплохо» ≠ сделка.
- **Бюджет**: есть ли деньги/готовность платить за это (не цену называть — понять реальность).
- **Срок**: есть ли триггер к действию сейчас, или «когда-нибудь» (последнее — не в активный пайплайн).
- **Решатель**: говорим с тем, кто решает/платит? Иначе — путь к нему.
Честная дисквалификация экономит обоим время. «Вам это не подходит» — валидный и правильный исход.

## 4. Возражения (по существу, этично)

- **«Дорого»** → не скидка первым ходом (Iron Law/анти-паттерн). Вернуть к цене проблемы из
  discovery: с чем сравнивают, какова стоимость бездействия. Если не по бюджету честно — отпустить.
- **«Мне надо подумать»** → мягко выяснить, о чём именно: цена / доверие / приоритет / решатель.
  Согласовать конкретный следующий шаг с датой, не «сам напишу когда-нибудь».
- **«У конкурента X»** → без очернения. Честно о своих сильных/слабых сторонах (Iron Law 2), где
  вы реально лучше для ЕГО job, а где нет. Если конкурент подходит ему больше — сказать это.

Никакой ложной срочности («только сегодня»), вины или давления. Возражение — это запрос информации.

## 5. Стадии пайплайна и конверсии

Lead → Qualified → Discovery done → Proposal → Closed (won/lost). На каждой — один явный next step
с датой. Мерить конверсию между стадиями по своим реальным данным (`nabu-domain`), чтобы видеть,
ГДЕ течёт. Сделка без next step дольше срока → пометить lost честно, не раздувать пайплайн иллюзией.
