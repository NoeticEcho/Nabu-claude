# Маркетинг для solo-фаундера (доменный пак)

Тонкий вход; глубина — в `references/`. Пак-эталон формы (SKILL.md ≤100 строк + references),
остальные домены получают паки по мере реальной потребности, не заранее.

## Iron Laws (незыблемо)

1. **Не выдумывать метрики, отзывы и «успешные кейсы»** — только реальные данные пользователя
   (`nabu-analytics`) или честное «данных нет, вот как собрать».
2. **AI-контент помечается** как AI-generated там, где площадка/закон этого требуют.
3. **Никакого спама и dark patterns** — ни в рекомендациях, ни в черновиках (fake scarcity,
   скрытые подписки, накрутка).
4. **Реклама регулируемых ниш** (медицина, финансы, дети) → юрист, не догадки.
5. **Числа волатильных платформ** (лимиты API, цены Ads, алгоритмы) не запекать — проверять
   через context7/веб на момент ответа.

## Когда что читать

| Задача | Файл |
|---|---|
| Стратегия: позиционирование, ICP, выбор каналов | `references/playbook.md` |
| Запуск кампании/контента: пошаговые чек-листы | `references/checklists.md` |
| Диагностика «почему не растёт» + типовые ошибки | `references/pitfalls.md` |

## Мини-цикл (по умолчанию)

1. Понять стадию: до product-market fit → эксперименты дешёвыми каналами; после → масштабирование
   работающего. Не наоборот.
2. Один ICP → одно обещание → один канал → одна метрика успеха. Расширение — после сигнала.
3. Каждая рекомендация — как **эксперимент**: гипотеза, метрика, срок, kill-критерий. Фиксировать
   через `nabu-domain.add_task` / `nabu-improve.record_recommendation`.

## Интеграция с Nabu

Позиции в Совет — через агента `marketing` (канон `agents/marketing.md`). Реальные метрики —
`nabu-analytics.list_metrics`/`aggregate_metric`. Рынок/SEO-данные — коннекторы
(`docs/INTEGRATIONS.md`: Open PageRank, PageSpeed, Dev.to). Черновики текстов — агент `copywriting`.
