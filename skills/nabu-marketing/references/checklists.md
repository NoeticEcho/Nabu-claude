# Чек-листы

## Запуск маркетингового эксперимента
- [ ] Гипотеза: «[канал/сообщение] приведёт [N ICP] за [срок], потому что [причина]»
- [ ] Одна метрика успеха + порог (kill-критерий) ДО старта
- [ ] Бюджет (деньги И часы) зафиксирован
- [ ] Лендинг/оффер отвечает «что/для кого/почему не X» за 10 сек
- [ ] Трекинг стоит (UTM/цели), первый замер снят
- [ ] Записано: `nabu-improve.record_recommendation` (узнаем исход, учимся)

## Лендинг (проверка перед трафиком)
- [ ] Заголовок = боль/результат ICP, не название продукта
- [ ] 1 CTA (не три), видим без скролла
- [ ] Социальное доказательство честное (нет отзывов — нет отзывов; Iron Law 1)
- [ ] Скорость: PageSpeed ≥ 80 mobile (коннектор PageSpeed из INTEGRATIONS)
- [ ] Ответ на топ-возражение прямо на странице

## Контент-цикл недели (solo)
- [ ] 1 столп-материал по реальной боли (из разговоров/сообществ, не из головы)
- [ ] 3-5 нарезок под площадки, где живёт ICP
- [ ] 80% времени — дистрибуция и разговоры, не полировка
- [ ] Ответы на КАЖДЫЙ комментарий/DM в первые 24ч
- [ ] Метрики недели → `nabu-domain`/`nabu-analytics` (что записываем — то растёт)

## Ежемесячная ревизия каналов
- [ ] По каждому активному каналу: контакты ICP / конверсия / CAC / часы
- [ ] Один худший канал остановить ИЛИ один эксперимент добавить (не оба)
- [ ] LTV/CAC пересчитан по свежим когортам
- [ ] Выводы — эпизодом в память (`remember_episode`), решения — задачами
