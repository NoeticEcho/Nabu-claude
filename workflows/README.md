# Nabu workflows (опциональные ассеты)

**Важно:** workflows в Claude Code — это JS-скрипты оркестрации (`.claude/workflows/*.js`),
**SDK/CLI-only**. Плагин их не поставляет и не устанавливает автоматически. Это ассеты для
power-users: скопируй нужный в `.claude/workflows/` (или запусти инструментом Workflow), и он
детерминированно оркестрирует десятки субагентов, держа промежуточные результаты вне контекста.

Когда использовать workflow вместо Совета/скиллов: **массовые/пакетные операции** (разобрать сотни
файлов, аудит всех сфер, сверка большого корпуса), где нужны детерминированный цикл, параллелизм и
воспроизводимость. Для интерактивного «спроси Совет» — используй `/nabu-council` (Agent Team).

## Примеры в этой папке
- `inbox-triage.workflow.js` — массовый разбор горнила `00-inbox`: по каждому файлу параллельно
  классификация (домен/тип/приватность) → извлечение сущностей/фактов → предложение назначения
  в структуре workspace. Итог — сводка + план перемещений (применение — по approval пользователя).

## Запуск
Через инструмент Workflow (в сессии Claude Code) или сохранив в `.claude/workflows/` и вызвав
как `/inbox-triage`. Приватное обрабатывается локально; ничего необратимого без approval.
