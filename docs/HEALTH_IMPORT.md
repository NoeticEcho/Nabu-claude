# Импорт данных о здоровье

Модуль импорта здоровья в Nabu построен вокруг одного принципа: **файловые экспорты вендоров,
разбираемые локально**. Никаких облачных API, никакого OAuth, никаких токенов.

## Философия: файлы, а не OAuth

Облачные API здоровья либо закрыты, либо непригодны для приватного персонального инструмента:

- **Google Fit REST API** закрывается (Google переводит всё на Health Connect на устройстве);
  строить интеграцию на умирающем API — тупик.
- **Huawei Health / Garmin / Fitbit** отдают данные через API только партнёрам по договору —
  обычному пользователю доступа нет.
- **OAuth-поток** означал бы, что данные о здоровье проходят через сторонний сервер и токены —
  это ровно то, чего инвариант приватности Nabu не допускает (медицина по умолчанию `private`).

Зато **экспорт файла есть у каждого вендора** (требование GDPR/«портируемость данных»): пользователь
выгружает свой архив, а Nabu разбирает его **на 100% локально** и складывает точки в общие
`metric_series` / `metric_values`. Экспорт приватнее API и работает у всех без исключения.

## Как экспортировать (по вендорам)

### Apple Health (iPhone)

1. Приложение «Здоровье» → аватар профиля (справа вверху) → **Экспортировать медданные**.
2. Получите `export.zip`, распакуйте → внутри `export.xml`.
3. Передайте `export.xml` в Nabu (файл может быть 100 МБ+ — разбирается потоково).

Импортируются: шаги, пульс, вес, активные калории, дистанция, сон (часы фаз Asleep).

### Google Fit (через Takeout)

1. Откройте <https://takeout.google.com> → снимите все галочки → отметьте только **Fit**.
2. Внутри Fit выберите **«Ежедневные показатели активности»** (Daily activity metrics), формат CSV.
3. Экспортируйте архив, распакуйте → `Daily activity metrics.csv`.

### Health Connect (Android)

Настройки телефона → **Health Connect** → **Данные и доступ** → **Экспорт данных**. Экспорт
приходит в CSV; если структура совпадает с Google Fit-полями — определится автоматически, иначе
сконвертируйте в generic CSV (ниже).

### Huawei Health

«Я» → **Настройки** → **Конфиденциальность** → **Запрос данных**. Через некоторое время придёт
ссылка на ZIP с CSV-файлами. **Формат Huawei нестабилен и не парсится автоматически** — переложите
нужные показатели в generic CSV (ниже) вручную или скриптом.

### Fitbit / Garmin

- **Fitbit**: аккаунт на fitbit.com → Settings → **Export Your Account Archive** (CSV).
- **Garmin**: Garmin Connect → Account → **Export Your Data** (CSV).

Оба отдают собственные CSV-раскладки — приведите к generic CSV.

## Формат generic CSV

Универсальный формат для любого источника, который не распознаётся напрямую. Заголовок
обязателен; порядок колонок значения не имеет, распознаётся по именам:

```csv
date,metric,value,unit
2026-07-01,steps,9000,count
2026-07-01,weight,81.5,kg
2026-07-01,sleep,7.5,h
2026-07-02,heart_rate,64,count/min
```

- `date` — ISO (`2026-07-01T08:00:00+03:00`) или голая дата `YYYY-MM-DD`.
- `metric` — имя ряда (напр. `steps`, `weight`, `heart_rate`, `sleep`, `active_energy`, `distance`).
- `value` — число.
- `unit` — необязательная колонка (единица ряда).

## Команда

```
nabu import-health <файл> [--source <имя>]
```

- Формат определяется автоматически (Apple XML / Google Fit CSV / generic CSV).
- `--source` — метка происхождения; в БД пишется как `import:<имя>` в `metric_values.source`
  (по умолчанию — вендор из автоопределения).
- Импорт **идемпотентен**: повторный запуск того же файла ничего не дублирует (dedup по
  ряду + времени + значению). Печатает `inserted` (новых точек), `deduped` (уже были),
  `series` (затронутые ряды).

Сырые точки складываются как есть — агрегация (сумма шагов за день и т.п.) не выполняется на
импорте, это задача аналитики.

## Что дальше

- **nabu-analytics**: forecast / trends по рядам здоровья (веса, шагов, сна).
- **Министр health**: интерпретация динамики в рамках границ компетенции (информация и
  структура, НЕ медицинский совет — см. `SAFETY.md`).
- **Дашборд**: визуализация рядов здоровья рядом с прочими сферами жизни.

## Честные ограничения

- **Huawei-формат не парсится автоматически** — его CSV-раскладка нестабильна между версиями
  приложения; путь через generic CSV.
- Единицы берутся из экспорта/маппинга «как есть»: Apple обычно отдаёт метрические (кг, км),
  но если у вас настроены имперские — конвертируйте до импорта либо через generic CSV с нужным `unit`.
- Импорт добавляет точки, но не удаляет ранее импортированные — чтобы «переимпортировать
  заново», данные ряда нужно чистить отдельно.
