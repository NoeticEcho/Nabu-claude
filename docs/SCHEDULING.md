# Расписание и циклы в Nabu-claude

Плагин Claude Code **не может напрямую поставлять cron** (см. официальные доки). Nabu использует
три штатных механизма Claude Code + опциональный системный cron. Управление — через `/nabu-cron`.

## Механизмы

| Механизм | Когда работает | Мин. интервал | Как включить |
|---|---|---|---|
| **Cloud routines** (`/schedule`) | всегда (облако Anthropic), без открытой сессии | 1 час | `/schedule "every day 9am" <промпт>` |
| **Сессионный `/loop`** | пока открыта сессия | 1 мин | `/loop 30m <промпт>` или `/loop` (само-темп, читает `.claude/loop.md`) |
| **Системный cron** (опц.) | всегда (ваша машина), headless | 1 мин | `scripts/install-cron.sh <job> "<cron>"` |
| **CronCreate/List/Delete** | сессионные напоминания | — | «напомни через 4 часа …» |

Шаблон для `/loop` — `assets/loop.md` (скопировать в `.claude/loop.md`).

## Проактивные задачи Nabu (каталог)
consolidate (ежедн.), digest (ежедн./еженед.), research (еженед.), scout (еженед.),
feedback (2-3×/нед), metrics (ежедн.). Подробности и команды — в `/nabu-cron`.

## Goals (`/goal`) — завершение по условию
Для многошаговых операций Nabu, где важен результат, а не время. Примеры:
- `/goal горнило 00-inbox разобрано: все файлы проиндексированы или перемещены, или стоп после 20 ходов`
- `/goal Совет вынес синтез с trade-off'ами и critic дал pass, или стоп после 15 ходов`
- `/goal все system_task со статусом open и priority=high обработаны, или стоп после 10 ходов`

`/goal` — сессионный, ставится пользователем или предлагается скиллом; Stop-hook может проверять
условия Nabu. Оценивает завершение быстрая модель по тексту диалога — формулируй проверяемо.

## Циклы (loops) — паттерн act → verify → correct
Итеративные агенты (researcher, effectiveness-evaluator) работают ограниченно (`maxTurns` в
субагенте) по паттерну: действие → проверка (read-only субагент) → коррекция. Непрерывный 24/7-фон
вне сессии — через cloud routines или (для тяжёлого) отдельный SDK-компаньон (см. ниже).

## Опция: SDK-компаньон для настоящего фона
Для 24/7 без сессии (непрерывный аудит памяти, метрики, авто-рефайн) можно вынести часть в
отдельный процесс на Claude Agent SDK, разделяющий ту же БД (`nabu_dev` + TypeDB). Плагин остаётся
интерактивным ядром; тяжёлый фон делегируется. Пока не входит в плагин — задел на будущее.
