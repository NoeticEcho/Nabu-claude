# Agent Teams в Nabu — Совет как команда агентов

Совет Nabu работает как **Agent Team**: релевантные министры спавнятся тиммейтами, обсуждают
кросс-доменные конфликты напрямую (SendMessage) и координируются общим task-list. Это ближе к
живому совещанию, чем последовательный сбор мнений.

## Включение (обязательно для team-режима)
Agent Teams — экспериментальная возможность Claude Code (opt-in). В `~/.claude/settings.json`
(или `.claude/settings.json` проекта):

```json
{
  "env": { "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1" },
  "teammateMode": "in-process"
}
```
`teammateMode`: `in-process` (по умолчанию, надёжно) · `tmux`/`iterm2` (видимые панели) · `auto`.

Если флаг НЕ включён — Совет автоматически работает в **fallback**: тот же протокол через
параллельный Task-диспатч + `deliberation`-буфер. Функциональность сохраняется; теряется только
прямое peer-обсуждение между министрами.

## Как это работает в Nabu
- **Lead** — адъютант (skill `nabu-orchestrator`), с ним общается пользователь.
- **Тиммейты** — министры (субагенты `agents/health.md` … `admin.md`) + `council` + `critic`.
  Тиммейт наследует от субагент-дефиниции `model` и (если заданы) `tools` + системный промпт
  (тело `.md`). Поля `skills`/`mcpServers` субагента тиммейтам НЕ передаются, но MCP-серверы
  плагина (`nabu-council`, `nabu-memory`, …) доступны как проектные — тиммейты ими пользуются.
- **Общая память** — не общий контекст, а общее состояние: task-list команды (координация) +
  `deliberation`-буфер в БД (позиции, синтез — durable-запись для памяти/аудита).
- **Общение** — `SendMessage` между тиммейтами (кросс-доменные зависимости), финальный синтез —
  тиммейтом `council`, проверка — `critic`. Итог собирает lead.
- **Реализовано в дефинициях**: системные промпты министров (`agents/*.md`), `council`, `critic`,
  `decision-maker` содержат секцию «Режим команды (Agent Teams)» — как координироваться через
  SendMessage/task-list и что durable-позиция всё равно пишется в буфер. Так тиммейт не деградирует
  до буфер-only поведения. При выключенных Teams та же секция описывает fallback.

## Границы
- ≤ `config.council.max_ministers_per_query` министров на совещание (бюджет).
- Высокорисковое из синтеза — через approval (`nabu-memory.request_approval`).
- Приватное — локально; в team-режиме тоже (тиммейты — тот же проект/окружение).

## Запуск
`/nabu-council <вопрос>` — адъютант поднимает Совет в team-режиме (или fallback). Явно про режим
можно спросить: «созвать Совет командой».
