# RPG / XP — геймификация Nabu

Лёгкий слой признания усилий поверх реальной жизни пользователя. Прогресс — из **настоящих**
событий (закрытые задачи, отмеченные привычки, достигнутые цели), не из накрутки. Правила
прозрачны, детерминированы и объяснимы: каждый XP имеет понятную причину и величину. Геймификация —
**поддержка, не давление** (инвариант #5, `SAFETY.md`): без FOMO, без шейминга, без dark patterns.

Реализация: чистые функции — `lib/src/rpg.ts`; начисления — `DomainRepository`
(`lib/src/repositories/domain.ts`); tools — `mcp/domain-server` (`get_character`, `award_xp`,
`update_task_status`, `log_habit`, `update_goal_status`). Хранение — `character_sheet` (8 колонок
`*_xp`) и журнал `xp_ledger` (по строке на каждое начисление, поле `reason`).

## 8 атрибутов

| Атрибут      | Смысл                                   | Сфера жизни           |
|--------------|-----------------------------------------|-----------------------|
| `intellect`  | знания, обучение, решение задач          | learning              |
| `wisdom`     | рефлексия, инсайты, зрелость суждений     | mind                  |
| `creativity` | творчество, созидание нового             | creative              |
| `discipline` | доведение дел до конца, регулярность      | work, admin (+ база)  |
| `vitality`   | здоровье, энергия, быт                   | health, lifestyle     |
| `resilience` | стойкость, преодоление, рост             | growth                |
| `sociality`  | связи, отношения, общение                | relationships         |
| `wealth`     | финансы, ресурсы, благосостояние          | finance               |

Маппинг сферы → атрибут: `domainToAttribute(domain)`. Неизвестный/пустой домен → `discipline`
(«просто сделал дело»).

## Детерминированные начисления (авто, без LLM)

Происходят сами при действии через `DomainRepository`; каждое пишется в `xp_ledger` с причиной.

| Событие                              | Атрибут              | XP                                    |
|--------------------------------------|----------------------|---------------------------------------|
| Закрыта задача (`update_task_status`)| `discipline`         | high **+15** / normal **+10** / low **+5** |
| …и была закрыта в срок (due_date)    | `discipline`         | бонус **+5** (`ONTIME_BONUS`)         |
| Отмечена привычка `done` (`log_habit`)| `discipline`        | **+5** (`HABIT_DISCIPLINE_XP`)        |
| …та же привычка, атрибут её домена   | домен привычки       | **+3** (`HABIT_DOMAIN_XP`)            |
| Достигнута цель (`update_goal_status`)| домен цели (деф. growth) | **+50** (`GOAL_XP`)              |

Идемпотентность: XP за задачу начисляется только на реальном переходе в `done` (повторная отметка
`done→done` не даёт дублей); за привычку/штраф — один раз в день на одинаковый статус.
`planned-skip` и `skipped` не сбивают серию и **не** наказываются (осознанный пропуск — не провал).

> Серия по метрике (`metric streak`) → `resilience` — правило есть, но начисляется адъютантом через
> `award_xp` по данным `nabu-analytics` (расчёт серии — в аналитике), а не в `log_metric`.

## Качественные начисления (agent-judged, `award_xp`)

За то, что не ловится детерминированно. Адъютант начисляет **скупо и честно**, с обязательной
причиной. Не дублировать то, что уже начислено автоматически.

| Что произошло                          | Атрибут      | Диапазон |
|----------------------------------------|--------------|----------|
| Инсайт / глубокая рефлексия             | `wisdom`     | 5–25     |
| Творческая работа                       | `creativity` | 5–25     |
| Живая социальная связь                  | `sociality`  | 5–25     |
| Преодоление трудности / срыва           | `resilience` | 5–25     |

Рубрика величины: **мелкое 5–10** (короткая мысль, набросок, тёплый разговор); **заметное 10–15**
(проработанная идея, содержательная встреча); **крупное 15–25** (прорыв, значимая работа, реальное
восстановление после спада). Потолок разового начисления — **25** (`QUALITATIVE_CAP`), чтобы не
инфлировать награды. `award_xp` в MCP жёстко ограничивает `amount` этим потолком.

## Штрафы (мягкие, объяснимые)

Единственный штраф: **сорванная серия привычки** → `discipline` **−5** (`HABIT_MISS_PENALTY`),
через `log_habit` со статусом `missed`. Атрибут **никогда не уходит в минус** — пол 0 гарантирован
на уровне записи (`greatest(0, …)`). Формулировка нейтральная («серия прервалась»), без шейминга.

**Бросить цель — штрафа НЕТ.** Отказ от цели — честное, взрослое решение, а не провал.

## Формулы уровней

Рост по квадратному корню (замедляется — честно, без бесконечной эскалации):

```
уровень атрибута = floor( sqrt( attr_xp / 50 ) )
общий уровень    = floor( sqrt( total_xp / 100 ) )   // total_xp — сумма по всем 8 атрибутам
```

Пороги атрибута: 0→ур.0, **50→ур.1**, **200→ур.2**, 450→ур.3, 800→ур.4 (порог ур.L = `50·L²`).
Пороги общего: 0→ур.0, **100→ур.1**, **400→ур.2**, 900→ур.3 (порог ур.L = `100·L²`).

`xpToNext(xp, "attribute"|"overall")` — сколько XP до следующего уровня (для показа прогресса).
`characterSummary(sheet)` собирает богатый лист: `{ level, total, toNext, attrs:[{name, xp, level,
toNext}] }`. Его отдаёт tool `get_character`.

## Инварианты

- Начислять только за **реально произошедшие** события из состояния/памяти — не выдумывать активность.
- Правила **прозрачны**: критерий и величина видимы; никакой скрытой механики, лутбоксов, «почти выиграл».
- Не поощрять сверхзависимость и гринд; XP — признание усилий, не крючок.
- Приватность: `private`/`vault`-данные не уходят в облако/логи; арифметика XP — локально, правилами.
